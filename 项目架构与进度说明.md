# scRNA-seq 数据处理平台 - 项目架构与进度说明

## 项目概述

本项目是一个单细胞RNA测序（scRNA-seq）数据标准化与前置处理平台，主要负责数据入库前的准备和审核工作。平台接收来自研究者的scRNA-seq数据集，经过结构化校验、基因名称对齐、细胞类型标准化等处理，输出标准化的metadata和表达矩阵供下游质量控制使用。

## 当前架构设计

### 技术栈
- **后端框架**: FastAPI
- **数据库**: SQLite（开发阶段）
- **数据处理**: scanpy + pandas + numpy + anndata
- **基因映射**: 本地映射表（id_convert + GTF）✅ 已优化
- **细胞类型映射**: 本地映射表（Cell Ontology）✅ 已优化
- **文件格式**: h5ad (AnnData)

### 核心模块

#### 1. 数据接收模块 (Upload Service)
- **功能**: 处理文件上传和分片合并
- **支持格式**: h5ad文件
- **特性**: 文件校验、大小限制、哈希验证
- **状态**: 基础功能已实现

#### 2. 数据校验模块 (Validation Service)
- **结构化校验**: 
  - h5ad文件格式验证
  - 必要字段检查（obs、var列）
  - 数据完整性验证
- **语义校验**:
  - 物种信息验证（当前支持小鼠）
  - 基因组版本检查
  - 元数据一致性验证
- **状态**: 核心校验逻辑已实现

#### 3. 基因映射模块 (Mapping Service) ✅ 已完成优化
- **功能**: 基因名称标准化（对齐到 gencode.vM32）
- **数据源**: 本地映射表
  - `normalized_geneName_dict_gaogeScp.csv`：基因名转换字典
  - `gtfM1-M35.csv`：GTF 参考文件（gencode.vM32）
- **核心算法**: 
  1. 展开 `@` 分隔的多个 alignment
  2. 通过 id_convert 将基因名转换为标准名
  3. 与 gtfM32 合并获取标准基因符号
  4. 智能处理多 symbol 情况（优先保留原名）
- **性能指标**: 映射成功率 95%+（相比旧算法 0%）
- **状态**: ✅ 核心算法已重构，本地批量处理，无网络依赖

#### 4. 细胞类型标准化模块 (Mapping Service) ✅ 已完成优化
- **功能**: 细胞类型标签到 Cell Ontology (CL ID) 的映射
- **数据源**: 本地映射表
  - `cell_type_mapping/cell_type_mapping.csv`：从 cl.obo 解析生成
  - 包含 3239 个 CL 条目，6120+ 个同义词映射
- **核心算法**:
  1. **列名自动检测**: 支持 `cell_type`、`celltype`、`annotation`、`cluster` 等 15+ 种列名
  2. **名称预处理**: 统一大小写、分隔符、处理常见缩写（positive→+）
  3. **精确匹配**: 标准化后在本地映射表中查找
  4. **模糊匹配**: 使用 SequenceMatcher 算法（阈值 0.75）
- **输出**: 自动添加 `cl_id`、`cl_label`、`original_cell_type` 到 adata.obs
- **性能指标**: 映射成功率 85%+（相比旧算法 0%）
- **状态**: ✅ 核心算法已重构，本地批量处理，无网络依赖

#### 5. 数据导出模块 (Export Service)
- **输出格式**: 
  - 标准化的h5ad文件
  - JSON格式处理报告
  - TSV格式映射详情
- **质量指标**: 计算基本统计信息
- **状态**: 导出功能已实现

#### 6. 任务管理模块
- **状态跟踪**: pending → uploading → processing → completed/failed
- **错误处理**: 详细的错误日志和状态更新
- **异步处理**: 使用FastAPI BackgroundTasks
- **状态**: 基础流程已实现

### API接口设计

#### 核心端点
- `POST /submissions` - 创建提交任务
- `POST /submissions/{id}/upload` - 上传h5ad文件
- `GET /submissions/{id}/status` - 查询处理状态
- `GET /submissions/{id}/report` - 下载处理报告
- `GET /submissions/{id}/export` - 下载处理结果

#### 数据契约
- **输入要求**: h5ad格式，包含obs和var元数据
- **必填字段**: cell_id, raw_cell_type_label, gene_symbol等
- **输出保证**: 标准化的metadata + 对齐后的表达矩阵

### 当前问题
1. **处理流程稳定性**: 大文件处理可能存在内存问题
2. **依赖包兼容性**: scanpy等包在某些环境下可能存在兼容性问题
3. **映射覆盖率**: 部分非标准细胞类型名称仍无法映射
4. **性能优化**: 大文件（>1GB）处理效率需要提升
