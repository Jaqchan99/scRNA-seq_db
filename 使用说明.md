# scRNA-seq 数据标准化平台 — 使用说明

本文档说明如何在本地安装、启动并复用本仓库中的 **Web 平台** 与 **命令行工具**。基因参考表需按需自行准备（见下文）。

## 一、环境要求

- Python 3.10+（建议 3.10 或 3.11）
- Windows / macOS / Linux 均可

## 二、安装依赖

在项目根目录执行：

```bash
pip install -r requirements.txt
```

若仅运行 **细胞类型映射** 相关功能，核心依赖为 `pandas`、`anndata` 等；若使用 **Web 后端** 或 **scanpy 读写 h5ad**，需完整安装 `requirements.txt`。

## 三、目录说明（克隆后你会看到）

| 路径 | 说明 |
|------|------|
| `src/` | 后端：FastAPI 应用与服务模块（`main.py` 为入口） |
| `scripts/` | 命令行：批量检查、benchmark 提取、评估、增量更新 SOP 等 |
| `frontend/` | 前端静态页面（HTML/CSS/JS） |
| `cell_type_mapping/cell_type_mapping.csv` | 细胞类型本地映射表（Cell Ontology 同义词表，运行映射所必需） |
| `cell_type_mapping/parse_cl_obo.py` | 从 OBO 生成/更新映射表的辅助脚本（可选） |

如需重新生成 `cell_type_mapping.csv`，请在本地下载 Cell Ontology 的 `cl.obo`（例如 [CL OBO release](http://purl.obolibrary.org/obo/cl.obo)）放到 `cell_type_mapping/cl.obo`，再运行 `parse_cl_obo.py`。

以下目录**默认不在仓库中**（或已被 `.gitignore` 忽略），用于本地数据与跑批结果，需自行创建：

| 路径 | 说明 |
|------|------|
| `data/raw/` | 基因映射用参考文件（见下文「基因参考文件」） |
| `data/h5ad/` | 本地存放待处理的 `.h5ad` |
| `data/benchmark/`、`results/` 等 | 评估与中间结果，按需本地生成 |

## 四、基因参考文件（离线基因名规范化）

`src/mapping_service.py` 默认从以下路径读取小鼠基因转换字典与 GTF 衍生表（相对于**项目根目录**）：

- `data/raw/normalized_geneName_dict_gaogeScp.csv`
- `data/raw/gtfM1-M35.csv`

**本仓库不附带这些大文件。** 使用完整「基因映射」流程前，请将上述文件放到 `data/raw/`。若你暂时没有这些文件，仍可尝试 **校验、细胞类型映射** 等步骤，但 **基因符号规范化** 会报错或无法达到预期。

可选：在 `data/raw/` 中保留 `jupyter_gene_mapping.txt` 等自用脚本时，请注意其中路径需与当前目录结构一致。

## 五、启动 Web 平台（推荐）

在项目根目录执行：

```bash
python scripts/start_platform.py
```

- **后端 API**：`http://localhost:8100`（与 `frontend/script.js` 中 `API_BASE_URL` 一致）
- **前端页面**：`http://localhost:8080`（默认；若端口占用，脚本会尝试递增端口，请以终端输出为准）

浏览器打开前端地址，按界面步骤：创建任务 → 上传 `.h5ad` → 校验 → 映射 → 导出。

### 仅启动后端（调试）

```bash
python src/main.py
```

### 仅启动前端静态服务

```bash
cd frontend
python -m http.server 8080
```

此时请确认 `script.js` 中的 `API_BASE_URL` 与后端端口一致（默认 `8100`）。

## 六、常用 API 示例

以下假设后端运行在 `http://localhost:8100`。

```bash
# 创建提交
curl -X POST "http://localhost:8100/submissions"

# 上传 h5ad（将 {id} 换为返回的 submission_id）
curl -X POST "http://localhost:8100/submissions/{id}/upload" -F "file=@/path/to/your.h5ad"

# 查询状态
curl "http://localhost:8100/submissions/{id}/status"
```

具体路由以 `src/main.py` 中定义为准。

## 七、命令行工具（脚本）

均在**项目根目录**下执行，便于相对路径解析。

### 7.1 批量扫描 h5ad，生成数据集列配置

```bash
python scripts/batch_inspect.py <h5ad所在目录>
```

默认输出：`configs/dataset_config.csv`（若本地无 `configs` 目录，请先创建或指定 `--output`）。

### 7.2 从 h5ad 提取 benchmark 标签

```bash
python scripts/extract_benchmark_labels.py <h5ad目录或路径列表.txt> --config configs/dataset_config.csv
```

默认输出：`data/benchmark/benchmark_labels.csv`（需已存在 `data/benchmark/` 或脚本会自动创建父目录，视脚本版本而定）。

### 7.3 评估映射效果（本地 / CellOntology 等）

```bash
python scripts/evaluate_mapping.py --method local
```

默认 benchmark：`data/benchmark/benchmark_labels.csv`；默认结果目录：`results/eval/`。若无 benchmark 文件，请先完成提取步骤或自行指定 `--benchmark`。

### 7.4 ZOOMA 在线评估（需网络）

```bash
python scripts/evaluate_zooma.py
```

### 7.5 增量更新 SOP（扩展 lookup 表）

一体化流程入口：

```bash
# 首次：指定包含新 h5ad 的目录
python scripts/incremental_update_sop.py --h5ad_dir /path/to/h5ad_folder

# 断点续跑（状态文件在 results/logs/ 下）
python scripts/incremental_update_sop.py --resume results/logs/sop_state_YYYYMMDD_HHMMSS.json

# 从某一阶段开始
python scripts/incremental_update_sop.py --h5ad_dir /path/to/h5ad_folder --start_phase merge
```

阶段包括：`scan` → `extract` → `resolve_cl_label`（可选）→ `eval_before` → `merge` → `eval_after` → `coverage`。具体说明见 `scripts/incremental_update_sop.py` 文件顶部注释。

### 7.6 合并人工标注到映射表

```bash
python scripts/merge_lookup_table.py --new data/labels/unmapped_labels.csv --table cell_type_mapping/cell_type_mapping.csv
```
